Pharmacy System Update

May Go Home and Walk-In Process Updates

This guide explains what changed, what staff will see on screen, and what will happen when adding medicines for patients under the updated pharmacy workflow.

1 Main menu label changed from For Discharge to May Go Home.
2 The Order Type display was added to the May Go Home list.
3 Some added items may now be saved under the patient's latest Walk-In encounter.

What Users Will Notice

The experience stays familiar, but a few screens now show clearer information and follow updated saving rules.

Navigation

New Name in the Menu

The menu item formerly called For Discharge is now called May Go Home. This is only a label change to make the purpose easier to understand.

Patient List

Order Type is Visible

The May Go Home table now shows the same order-type indicators used in Prescription Ward: Basic, G24, and OR.

Item Adding

Items Can Still Be Added

Staff can continue adding items. The system now decides whether the item stays on the current encounter or is routed through the latest Walk-In encounter.

Behind the Scenes

Original Encounter is Remembered

Even when an item is saved under Walk-In, the system still remembers the original encounter so users can track where the request came from.

How the Updated Saving Rule Works

This is the main workflow change for end users.

1

User opens a patient

The user can open the patient from the May Go Home list or from the usual encounter transaction screen.

2

User adds an item

The user adds a drug or medicine just like before. No extra step is needed from the user.

3

System checks the bill status

If the encounter's bill status is 02 or 03, the item is saved under the patient's latest Walk-In encounter.

  • The original encounter is still linked for reference.
  • The item can still be seen from the original screen.
4

If bill status is not 02 or 03

The item stays on the current encounter, but the user is still allowed to add the item.

Quick Before and After

Before

If the patient's encounter was already billed with bill status 02 or 03, users could no longer add items in that encounter. To continue, they had to manually open the patient's Walk-In encounter, or create a new one, just to add another item. Also, prescribed items did not appear under the Walk-In encounter, which made the workflow harder to follow.

After

When bill status is 02 or 03, the system now automatically uses the latest Walk-In encounter in the background when adding items. Users no longer need to manually switch encounters just to continue adding medicines, and prescribed items can now be handled properly in this updated flow.

Simple Scenarios

  • If a patient appears under May Go Home, users will now see the order-type indicators in that list.
  • If a user adds an item and the bill status is 02 or 03, the item goes to the latest Walk-In encounter.
  • If a user adds an item and the bill status is anything else, the item stays on the current encounter.
  • Users do not need to manually decide where the item goes. The system handles that automatically.
